# CSS Skill Specification
File: css.md

## Purpose

CSS controls the visual presentation of elements defined in HTML (html.md).

CSS selectors must reference ONLY IDs and classes from the HTML file.
CSS must define ALL state classes that JavaScript (js.md) toggles.

---

# Design Philosophy — Uncodixify

The goal is UI that looks like it was designed by a human, not generated by AI.
Think Linear, Raycast, Stripe, GitHub. Functional, honest, calm.

## What to AVOID (banned AI-default patterns)

- Gradient backgrounds on buttons, cards, or body — use solid fills
- Oversized border-radius (over 10px on cards/panels, over 8px on buttons)
- Pill-shaped buttons (border-radius: 50px or 9999px)
- Gradient text (background-clip: text / -webkit-text-fill-color: transparent)
- Dramatic shadows (blur over 12px, colored shadows, glow effects)
- Glass/frosted panel effects (backdrop-filter, rgba white surfaces)
- Transform animations on hover (translateY, scale, translateX)
- Bounce or spring animations — use simple opacity/color transitions only
- Slide-in animations on modals — just show/hide with .hidden
- Blue-dominant dark modes with cyan accents (generic SaaS look)
- Eyebrow labels (uppercase + letter-spacing decorative text)
- Radial gradient backgrounds as decoration
- Multiple nested panel types with different corner radii
- `Segoe UI`, `Trebuchet MS`, `Arial`, `Inter`, `Roboto` as font choices
- Random color combinations — always pick from the approved palettes below

## What to DO instead

- Solid button fills, 6-8px radius max
- Cards: 1px solid border, 8px radius max, shadow max `0 2px 8px rgba(0,0,0,0.1)`
- Transitions: 150ms ease, opacity or background color only
- Headers: plain `h1`/`h2`, normal weight, no decorative treatment
- Modals: simple centered overlay, no animation, plain close button
- Spacing scale: 4 / 8 / 12 / 16 / 24 / 32px — no random gaps
- Borders: 1px solid, neutral color
- Typography: system-ui or a clean geometric sans (not the banned list above)

# Standard State Classes (Critical Cross-File Contract)

CSS MUST define these state classes. JS toggles them, CSS styles them.

REQUIRED — always include these in every stylesheet:

```css
.hidden {
  display: none !important;
}
```

`.active` — style varies per component:

```css
.nav-item.active {
  border-bottom: 2px solid var(--primary);
  font-weight: 600;
}
```

`.disabled`:

```css
.btn.disabled {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
}
```

CRITICAL: `.hidden { display: none !important; }` MUST be present in EVERY CSS file.
JS uses `classList.add("hidden")` / `classList.remove("hidden")`.

BANNED — never define these:
- `.is-open`, `.is-hidden`, `.is-visible`, `.show`, `.visible`, `.open`, `.closed`
- `.is-active`, `.is-disabled`, `.is-selected`

# Modal / Overlay Styling

Modals use the `.hidden` class for visibility. No animation on show/hide.

```css
.modal-overlay {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background: rgba(0, 0, 0, 0.5);
  display: flex;
  align-items: center;
  justify-content: center;
  z-index: 1000;
}

.modal-overlay.hidden {
  display: none !important;
}

.modal-content {
  background: var(--surface, #fff);
  border: 1px solid var(--border);
  border-radius: 8px;
  padding: 24px;
  max-width: 500px;
  width: 90%;
  max-height: 80vh;
  overflow-y: auto;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.15);
}
```

NO slide-in animations. NO opacity transitions. Just `.hidden` toggling.

# Button Classes (Standard)

Solid fills, 6-8px radius, simple hover (color shift only, no transform).

```css
.btn {
  padding: 8px 14px;
  border: none;
  border-radius: 6px;
  cursor: pointer;
  font-size: 0.875rem;
  font-weight: 500;
  transition: background 0.15s ease;
}

.btn-primary {
  background: var(--primary);
  color: #fff;
}

.btn-primary:hover {
  background: var(--primary-dark, color-mix(in srgb, var(--primary), #000 15%));
}

.btn-secondary {
  background: var(--surface);
  color: var(--text);
  border: 1px solid var(--border);
}

.btn-secondary:hover {
  background: var(--border);
}

.btn-danger, .btn-delete {
  background: var(--danger, #dc2626);
  color: #fff;
}

.btn-edit {
  background: var(--secondary);
  color: #fff;
}
```

NO gradient backgrounds. NO translateY on hover. NO pill shapes.

# Card Styling

Simple borders, subtle shadow, radius 8px max.

```css
.note-card {
  border: 1px solid var(--border);
  border-radius: 8px;
  padding: 16px;
  background: var(--surface);
  box-shadow: 0 1px 4px rgba(0, 0, 0, 0.08);
}
```

NO `transform: translateY(-5px)` on hover. NO colored shadows. NO blur over 8px.

# Dynamic Element Styling (Critical)

When JS creates elements dynamically, CSS must style the EXACT class names used.

Read the completed JS file for the actual class names in createElement calls.
The class names in HTML, JS, and CSS MUST all match exactly.

# Typography

```css
body {
  font-family: system-ui, -apple-system, sans-serif;
  font-size: 16px;
  line-height: 1.5;
  color: var(--text);
}
```

- Body text: 14-16px
- No mixed serif/sans combos
- No gradient text
- No uppercase + letter-spacing decorative headers
- `h1`/`h2` with plain weight and normal color — no special treatment

# Layout Guidelines

```css
/* Standard container */
.container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 24px;
}

/* Standard header */
.app-header {
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: 12px 24px;
  border-bottom: 1px solid var(--border);
  background: var(--surface);
}
```

- Sidebars: 240-260px fixed, solid background, `border-right: 1px solid var(--border)`
- No floating panels. No rounded outer shells on sidebars.
- Standard grid gaps: 16px or 24px
- Flexbox for 1D, Grid for 2D

---

# Spacing Scale

Use only these values: 4px, 8px, 12px, 16px, 24px, 32px, 48px.
No random gaps or padding values.

---

# Transitions

```css
transition: background 0.15s ease;
transition: color 0.15s ease;
transition: opacity 0.15s ease;
transition: border-color 0.15s ease;
```

- Max duration: 200ms
- Only on: background, color, opacity, border-color
- NO: transform, scale, translateX/Y, box-shadow transitions
- NO bounce, spring, or elastic easing

# Responsive Design

```css
@media (max-width: 768px) {
  .note-card-header {
    flex-direction: column;
    gap: 8px;
  }
}
```

Mobile collapse must remain functional — not just a tall stack of gray boxes.

# Integration Contract

## With HTML (html.md)
- Style ALL classes and IDs from the HTML file
- Do NOT create selectors for elements that don't exist in HTML
- Always define `.hidden { display: none !important; }`

## With JavaScript (js.md)
- Style every class JS toggles (hidden, active, disabled)
- Style every class JS uses in dynamic element creation
- Match class names EXACTLY — read the completed JS to verify

---

# Summary

CSS MUST:
- Pick a palette from the approved list (not invent colors)
- Use solid fills, not gradients on interactive elements
- Keep border-radius ≤ 8px on cards/panels, ≤ 6px on buttons
- Keep shadows subtle: max `0 2px 8px rgba(0,0,0,0.1)`
- Use 150ms ease transitions on color/opacity only
- Define `.hidden { display: none !important; }`
- Style all standard button classes
- Style all dynamic element classes from JS
- Style all IDs and classes in HTML
